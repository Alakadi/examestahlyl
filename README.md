# منصة اختبارات تحديد المستوى

منصة متطورة لاختبارات تحديد المستوى مع لوحة تحكم متقدمة للمسؤولين وواجهة تفاعلية للطلاب. تتضمن المنصة نظام ذكاء اصطناعي متقدم لتقديم شروحات مخصصة وتقييمات ذكية.

## المميزات الرئيسية

### للطلاب
- **اختبارات مجانية ومدفوعة**: الوصول المباشر للاختبارات المجانية أو إدخال كود للاختبارات المدفوعة
- **واجهة تفاعلية**: تجربة اختبار سلسة مع تغذية راجعة فورية
- **مساعد ذكاء اصطناعي**: اسأل الذكاء الاصطناعي لشرح الأسئلة والإجابات الخاطئة
- **تقييم شامل**: نتائج مفصلة مع تحليل أداء الأقسام والتوصيات التعليمية
- **شهادات**: الحصول على شهادات عند اجتياز الاختبارات

### للمسؤولين
- **إدارة المواد**: إنشاء وتعديل وحذف المواد الدراسية
- **إدارة الأقسام**: تنظيم الأسئلة في أقسام متعددة
- **إدارة الأسئلة**: إضافة أسئلة يدوياً أو استيراد من ملفات Excel
- **إدارة الاختبارات**: إنشاء اختبارات مع توزيع متساوي للأسئلة من الأقسام
- **إدارة الأكواد**: توليد وإدارة أكواد الوصول للاختبارات المدفوعة
- **إدارة التقييم**: نصوص تقييم ديناميكية بناءً على الدرجات والأقسام
- **تحليل الاستخدام**: إحصائيات شاملة عن استخدام الأكواد والاختبارات

## المتطلبات

- **Node.js**: الإصدار 18 أو أحدث
- **PostgreSQL**: قاعدة البيانات
- **pnpm**: مدير الحزم (مثبت مسبقاً في package.json)

## خطوات الإعداد

### 1. استنساخ المستودع

```bash
git clone https://github.com/Alakadi/examestahlyl.git
cd examestahlyl
```

### 2. تثبيت التبعيات

```bash
pnpm install
```

### 3. إعداد قاعدة البيانات

#### أ. إنشاء قاعدة البيانات

```bash
# تسجيل الدخول إلى PostgreSQL
psql -U postgres

# إنشاء قاعدة البيانات
CREATE DATABASE examestahlyl;

# الخروج
\q
```

#### ب. إنشاء ملف .env

أنشئ ملف `.env` في جذر المشروع بالمحتوى التالي:

```env
# قاعدة البيانات
DATABASE_URL=postgresql://postgres:password@localhost:5432/examestahlyl

# المصادقة
JWT_SECRET=your-super-secret-jwt-key-change-this-in-production

# البيئة
NODE_ENV=development

# المنفذ (اختياري)
PORT=3000
```

**ملاحظة**: استبدل `password` بكلمة المرور الفعلية لمستخدم PostgreSQL.

#### ج. تطبيق Schema على قاعدة البيانات

```bash
pnpm db:push
```

### 4. تشغيل المشروع

#### في بيئة التطوير

```bash
pnpm dev
```

المشروع سيكون متاحاً على: `http://localhost:3000`

#### في بيئة الإنتاج

```bash
pnpm build
pnpm start
```

## البنية البيانية

### المواد (Subjects)
- معرّف فريد
- العنوان
- الوصف
- الرمز التعبيري (Icon)
- اللون

### الأقسام (Sections)
- معرّف فريد
- معرّف المادة
- العنوان
- الوصف
- ترتيب العرض

### الأسئلة (Questions)
- معرّف فريد
- معرّف القسم
- نص السؤال
- الخيارات (4 خيارات على الأقل)
- الإجابة الصحيحة
- الشرح التوضيحي
- رابط الشرح
- نص للذكاء الاصطناعي

### الاختبارات (Exams)
- معرّف فريد
- معرّف المادة
- العنوان
- النوع (مجاني/مدفوع)
- عدد الأسئلة الكلي
- توزيع الأسئلة حسب الأقسام
- حد زمني (اختياري)
- درجة النجاح (اختياري)

### أكواد الاختبارات (Exam Codes)
- معرّف فريد
- الكود الفريد
- معرّف الاختبار
- عدد الاستخدامات القصوى (اختياري)
- تاريخ الانتهاء (اختياري)
- حالة التفعيل
- عدد الاستخدامات الحالية

### نتائج الاختبارات (Exam Results)
- معرّف فريد
- معرّف المستخدم
- معرّف الاختبار
- الدرجة الكلية
- درجات الأقسام
- الإجابات المفصلة
- وقت البدء والانتهاء

### نصوص التقييم (Assessment Texts)
- معرّف فريد
- معرّف الاختبار
- نطاق الدرجات (من - إلى)
- النص التقييمي
- شروط الأقسام (اختياري)

## API Endpoints

### المصادقة
- `GET /api/auth/me` - الحصول على بيانات المستخدم الحالي
- `POST /api/auth/logout` - تسجيل الخروج

### المواد
- `POST /api/subjects` - إنشاء مادة جديدة (Admin)
- `GET /api/subjects` - الحصول على جميع المواد
- `GET /api/subjects/:id` - الحصول على مادة محددة
- `DELETE /api/subjects/:id` - حذف مادة (Admin)

### الأقسام
- `POST /api/sections` - إنشاء قسم جديد (Admin)
- `GET /api/sections/subject/:subjectId` - الحصول على أقسام مادة
- `DELETE /api/sections/:id` - حذف قسم (Admin)

### الأسئلة
- `POST /api/questions` - إنشاء سؤال جديد (Admin)
- `GET /api/questions/section/:sectionId` - الحصول على أسئلة قسم
- `POST /api/questions/bulk-import` - استيراد أسئلة من Excel (Admin)
- `DELETE /api/questions/:id` - حذف سؤال (Admin)

### الاختبارات
- `POST /api/exams` - إنشاء اختبار جديد (Admin)
- `GET /api/exams/:id` - الحصول على اختبار محددة
- `GET /api/exams/subject/:subjectId` - الحصول على اختبارات مادة
- `GET /api/exams/:id/questions` - الحصول على أسئلة الاختبار
- `DELETE /api/exams/:id` - حذف اختبار (Admin)

### أكواد الاختبارات
- `POST /api/exam-codes` - إنشاء كود جديد (Admin)
- `GET /api/exam-codes/validate/:code` - التحقق من صحة الكود
- `GET /api/exam-codes/exam/:examId` - الحصول على أكواد اختبار (Admin)

### النتائج
- `POST /api/results/submit` - تسليم نتيجة اختبار
- `GET /api/results/user` - الحصول على نتائج المستخدم
- `GET /api/results/:id` - الحصول على نتيجة محددة

### التقييم
- `POST /api/assessment-texts` - إنشاء نص تقييم (Admin)
- `GET /api/assessment-texts/exam/:examId` - الحصول على نصوص التقييم

### الدردشة
- `POST /api/chat` - إرسال رسالة للذكاء الاصطناعي

## الميزات المتقدمة

### توزيع الأسئلة المتساوي
عند إنشاء اختبار، يتم تحديد عدد الأسئلة المطلوبة من كل قسم، ويتم اختيارها عشوائياً من الأسئلة المتاحة في كل قسم.

### استيراد الأسئلة من Excel
يمكن استيراد الأسئلة من ملف Excel يحتوي على:
- عمود الأسئلة
- أعمدة الخيارات (A, B, C, D)
- عمود الإجابة الصحيحة
- عمود الشرح (اختياري)

### مساعد الذكاء الاصطناعي
عند الإجابة الخاطئة على سؤال، يمكن للطالب فتح نافذة الدردشة والتحدث مع الذكاء الاصطناعي الذي يستقبل:
- نص السؤال
- الإجابة الصحيحة
- إجابة الطالب
- النص التوضيحي من المسؤول

### التقييم الديناميكي
يتم تقديم تقييم مخصص بناءً على:
- الدرجة الكلية (مثلاً: 80-100% → ممتاز)
- أداء الأقسام (مثلاً: ضعيف في قسم → رسالة توجيهية)
- دمج رسائل متعددة إذا كان الطالب ضعيفاً في أكثر من قسم

## استكشاف الأخطاء

### المشكلة: فشل الاتصال بقاعدة البيانات

**الحل**:
1. تأكد من تشغيل PostgreSQL
2. تحقق من `DATABASE_URL` في ملف `.env`
3. تأكد من أن المستخدم لديه صلاحيات كافية

### المشكلة: خطأ "Port already in use"

**الحل**:
```bash
# تغيير المنفذ
PORT=3001 pnpm dev

# أو إيقاف العملية التي تستخدم المنفذ
lsof -i :3000
kill -9 <PID>
```

### المشكلة: خطأ في الذكاء الاصطناعي

**الحل**:
1. تأكد من أن مفتاح API صحيح
2. تحقق من الاتصال بالإنترنت
3. تحقق من حدود الاستخدام
